# CFLIB is a small, simple, flexible and portable ANSI C Library to be used as
# configuration interface for user programs. The library builds and maintains a
# compact database structure consisting of a list of parameters with their name,
# content and some bytes of meta information about each parameter.
# 
# CFLIB targets the basic needs of technical, scientific or other programmers
# who want to spend minimal time on coding input, output, variable parsing,
# report generation and the like but still have a simple to use, reliable,
# flexible and portable configuration interface for their programs.
#

Essential Usage:

- Include include/cf.h
- Link lib<version>/libcf.a (or other library name)

Essential Building Example:

 cd lib
 gcc -c ../src/*.c
 ar -r libcf.a

Test:

 cd src/examples
 gcc cf_test.c -lcf -L ../../lib -o cf_test
 ./cf_test
 
Essential Makefile Example:

 make info help
 make clean package documentation

Advanced Building:

See files listed under "Library Building Support Files".

In the root directory you find the files: 

 makefile
 makefile.init
 makefile.defs
 makefile.targets
 [lib/makefile]

They are defining a framework that works together with generated makefiles
from GNU Make Build Environment based on Eclipse IDE Toolchains as well as
custom project makefiles.

All custom settings are in file

 makefile.projects or
 makefile.projects.<system>

Call make once to generate a makefile.projects[.<system>] if there is none.

Every Library Build and the Examples Build are performed by the makefile
in subdirectories with matching name and can be made by a

 make <build>
 
By default, <build> will be "liblin" for Linux, "libmgw" for Windows and
"lib" for any other (unrecognized) system.

The build files makefile, objects.mk, sources.mk, subdir.mk in subdirectories
other than "lib" have been generated by Eclipse tools and may need regeneration
or editing, when used.
